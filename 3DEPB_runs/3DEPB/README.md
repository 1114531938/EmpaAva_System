# 3D Emotional Phone Booth 产品介绍

## 产品定位

3D Emotional Phone Booth 是一个面向情绪陪伴、心理支持和沉浸式交流场景的数字人对话系统。用户登录后，可以选择不同身份的数字人，通过视频、音频和文字进行交流，并保存对话记录与录制视频。

产品目标是构建一个“可登录、可对话、可管理、可留档”的数字人情绪交互平台，为后续接入真实数字人算法、语音识别、语音合成和视频生成能力提供完整应用框架。

## 核心功能

### 1. 登录入口

主页面展示产品名称：

```text
3D Emotional Phone Booth
```

用户输入账号和密码后登录系统。系统目前已经接入后端账号体系，用户信息存储在 SQLite 数据库中，密码使用 PBKDF2 哈希存储。

默认账号：

- 管理员：admin / admin123
- 普通用户：user / user123

### 2. 数字人对话页面

对话页面参考腾讯会议式布局：

- 左侧：用户本地视频画面
- 中间：数字人展示区域
- 右侧：数字人身份选择、历史对话、录制视频、输入区

用户可以选择不同数字人身份：

- 情绪陪伴者
- 成长导师
- 亲密朋友
- 心理教练

用户输入内容后，前端调用后端数字人回复接口：

```text
POST /api/avatar/respond
```

当前后端返回 mock 回复，并预留：

- replyText：数字人回复文本
- videoUrl：算法生成的视频地址
- audioUrl：算法生成的音频地址

后续算法团队只需要替换后端接口内部逻辑，即可接入真实数字人生成能力。

### 3. 音视频控制

对话页面支持：

- 开关麦克风
- 开关摄像头
- 开始 / 停止录制
- 保存录制视频
- 结束对话

录制使用浏览器 MediaRecorder 完成。保存录制时，前端将视频 Blob 通过 FormData 上传到后端，后端保存到 recordings 目录，并写入数据库。

### 4. 历史对话记录

每次用户发送内容并获得数字人回复后，系统会自动保存对话记录。

历史记录包含：

- 用户
- 数字人身份
- 用户输入内容
- 数字人回复内容
- 创建时间

权限规则：

- 普通用户只能查看自己的历史对话
- 管理员可以查看全部用户的历史对话

接口：

```text
GET /api/conversations
```

### 5. 录制视频管理

用户可以将本次对话录制保存到后端。

录制记录包含：

- 用户
- 数字人身份
- 视频文件路径
- 文件大小
- 创建时间

权限规则：

- 普通用户只能查看自己的录制视频
- 管理员可以查看全部用户录制视频

接口：

```text
POST /api/recordings
GET /api/recordings
```

### 6. 用户账号管理

管理员可以进入用户管理页面，管理系统内所有账号。

支持功能：

- 查看所有用户
- 新增用户
- 修改用户密码
- 启用 / 禁用用户
- 删除用户

系统会阻止管理员删除或禁用当前登录的管理员账号。

## 用户流程

### 普通用户流程

1. 打开系统首页
2. 输入账号密码登录
3. 进入数字人对话页面
4. 选择喜欢的数字人身份
5. 开启摄像头和麦克风
6. 输入想说的话
7. 查看数字人回复
8. 可选择录制并保存视频
9. 查看自己的历史对话和录制视频

### 管理员流程

1. 使用管理员账号登录
2. 进入用户管理页面
3. 创建或维护用户账号
4. 返回对话页面
5. 查看全局历史对话和录制视频

## 技术架构

### 前端

当前前端为原生 HTML、CSS、JavaScript 实现，无需构建工具即可运行。

主要文件：

- index.html
- styles.css
- app.js

前端能力：

- Hash 路由
- 登录状态管理
- 摄像头 / 麦克风调用
- MediaRecorder 录制
- FormData 视频上传
- 数字人身份切换
- 历史对话展示
- 录制视频播放
- 用户管理交互

### 后端

后端使用 Python 标准库实现轻量服务。

主要文件：

- server.py

后端能力：

- 静态页面服务
- SQLite 数据库
- Cookie Session 鉴权
- PBKDF2 密码哈希
- 用户管理接口
- 数字人回复接口
- 历史对话接口
- 录制视频上传与访问

### 数据存储

SQLite 数据库位置：

```text
data/app.db
```

视频文件保存位置：

```text
recordings/
```

主要数据表：

- users
- sessions
- conversations
- recordings

## 后续规划

### 第一阶段：算法接入

将当前 mock 数字人回复替换为真实算法服务。

建议后端内部调用算法服务：

```text
用户输入文本 / 音频
-> 算法服务
-> 回复文本
-> 音频地址
-> 数字人视频地址
```

前端无需大改，只需要播放后端返回的 videoUrl 和 audioUrl。

### 第二阶段：语音交互

增加语音输入能力：

- 用户说话
- 浏览器录音
- 后端语音识别
- 转文字后发送给数字人接口

### 第三阶段：完整录制合成

当前录制主要保存用户摄像头流。后续可以升级为：

- 用户视频
- 数字人视频
- 双方音频
- 对话字幕

合成为完整对话视频。

### 第四阶段：后台增强

管理员后台可增加：

- 用户搜索
- 用户分页
- 历史对话检索
- 录制视频检索
- 数据导出
- 用户使用统计

## 产品价值

3D Emotional Phone Booth 的价值在于把数字人能力包装成一个完整可用的交互系统，而不是单一算法 demo。它同时覆盖了用户登录、对话交互、身份选择、录制留档、历史追踪和后台管理，为情绪陪伴、心理支持、教育陪练、客服模拟等场景提供了可扩展基础。
